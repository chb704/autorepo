# Design Vision

> This document is the standard the frontend Design Review grades against. The more
> specific and opinionated it is, the better the review works. Replace everything below
> with the real vision for your product.

## Principles

- The feel you're going for, in a few sentences. (Calm? Dense and powerful? Playful?)

## Typography

- Typefaces, scale, weight usage, line-height conventions.

## Color

- Palette and how each color is allowed to be used. Call out what's **not** allowed
  (one-off colors, ad-hoc hexes).

## Spacing & layout

- The spacing scale and grid rules. Consistency here is what keeps ten agents from
  producing a ten-personality UI.

## Components

- Reusable patterns that already exist. New work should compose these, not reinvent them.

## Interaction & motion

- How things should respond, animate, and transition.

## Anti-patterns

- Concrete things the review should reject on sight.
